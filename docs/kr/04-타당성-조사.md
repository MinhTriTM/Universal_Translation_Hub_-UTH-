# 타당성 조사
## Feasibility Study — Universal Translation Hub (UTH)

**버전:** 1.0
**날짜:** 2026-05-22
**저자:** Đoàn Minh Trí

---

## 1. 개요

이 문서는 UTH 프로젝트의 타당성을 4가지 측면에서 평가합니다: **기술적**, **재정적**, **일정**, **운영**.

---

## 2. 기술적 타당성 (Technical Feasibility)

### 2.1 핵심 기술 평가

| 요소 | 평가 | 상세 |
|------|------|------|
| **MiMo V2.5 API** | ✅ 타당 | OpenAI 호환, 문서 보유, Python SDK |
| **멀티 에이전트 아키텍처** | ✅ 타당 | 일반적인 패턴, Python 비동기 지원 우수 |
| **게임 텍스트 추출** | ✅ 이미 보유 | DichGame에서 12개 엔진 구현 완료 |
| **만화 OCR + 인페인팅** | ✅ 이미 보유 | manga-image-translator 안정적으로 작동 |
| **영화 STT + TTS** | ⚠️ 대부분 보유 | MIMO-AXON 베타, MiMo TTS 통합 필요 |
| **Translation Memory** | ✅ 단순 | SQLite, DichGame에 패턴 존재 |
| **음성 클론** | ⚠️ 확인 필요 | MiMo API 지원 여부에 따라, Edge-TTS 폴백 보유 |

### 2.2 기술 스택

```

├─────────────────────────┬───────────────────────┤

```

### 2.3 기술 역량 평가

| 기술 | 현재 보유 | 추가 필요 |
|------|-----------|-----------|
| Python 프로그래밍 | ✅ 숙련 | — |
| FastAPI/Web 개발 | ✅ 경험 보유 | — |
| AI/ML 통합 | ✅ 수행 경험 (Gemini, Dolphin) | MiMo API 세부 사항 |
| 게임 리버스 엔지니어링 | ✅ 12개 엔진 | 새 엔진 추가 |
| 이미지 처리 (CV) | ✅ manga-image-translator | — |
| 오디오 처리 (TTS/STT) | ⚠️ 기본 | 고급 오디오 동기화 |
| 멀티 에이전트 아키텍처 | ⚠️ 설계 완료 | 프로덕션 구현 |

### 2.4 기술 결론
> **타당** — 3개 프로젝트에서 이미 70% 코드 보유. 나머지는 주로 통합 + MiMo API 클라이언트.

## 3. 재정적 타당성 (Financial Feasibility)

### 3.1 MiMo API 비용

| 패키지 | 가격 | 크레딧 | 적합성 |
|--------|------|--------|--------|
| Lite | $15/월 | 50M | ❌ 너무 적음 |
| Pro | $50/월 | 400M | ⚠️ 개발용 충분, 프로덕션 부족 |
| **Max** | **$100/월** | **1.6B** | ✅ 전체 프로젝트에 충분 |

### 3.2 비용-효과 분석

**비용 (8개월):**

| 항목 | 비용 |
|------|------|
| MiMo Max (8개월) | $800 |
| 호스팅 (필요 시) | $0 (로컬 실행) |
| 도메인 (필요 시) | $12/년 |
| **합계** | **~$812** |

**효과:**

| 효과 | 가치 |
|------|------|
| 게임 번역 시간 절약 (1게임 = 수동 ~40시간 → 자동 2시간) | ~$2,000/게임 |
| 만화 번역 시간 절약 (1챕터 = 수동 ~8시간 → 30분) | ~$500/챕터 |
| 영화 더빙 시간 절약 (1화 = 수동 ~6시간 → 1시간) | ~$300/에피소드 |
| 오픈 소스 커뮤니티 가치 | 측정 불가 |

> **손익분기점:** 게임 1개 또는 만화 챕터 2개 번역만으로 투자금 회수 가능.

### 3.3 재정 결론
> **타당** — 저비용 ($100/월), 고효과. Max 패키지로 개발과 프로덕션 모두 충분.

## 4. 일정 타당성 (Schedule Feasibility)

### 4.1 일정 분석

| 단계 | 기간 | 주요 태스크 | 신뢰도 |
|------|------|-------------|--------|
| Phase 1 | 8주 | 기반 + MiMo 통합 | 90% |
| Phase 2 | 8주 | 파이프라인 통합 | 80% |
| Phase 3 | 8주 | 음성 + 품질 | 75% |
| Phase 4 | 4주 | 개선 + 릴리스 | 85% |
| Phase 5 | 4주 | 확장 + 최적화 | 70% |
| **합계** | **32주 (8개월)** | | **80%** |

### 4.2 크리티컬 패스

```
MiMo API Client → Director Agent → Router Agent → Pipeline Integration → QA Agent → Release
     (1주)           (1.5주)         (1주)            (8주)              (1.5주)    (0.5주)
```

**크리티컬 패스 = 12.5주 (3.1개월)** — MVP (게임 파이프라인만) 기준.

### 4.3 일정 결론
> **타당** — 현실적인 타임라인. 70% 코드 이미 보유, 대부분의 시간은 통합에 소요.

## 5. 운영 타당성 (Operational Feasibility)

### 5.1 목표 사용자

| 그룹 | 예상 수 | 사용 사례 |
|------|---------|-----------|
| 베트남 게이머 | ~500,000+ | 중국/일본 게임 베트남어 현지화 |
| 만화 독자 | ~1,000,000+ | 베트남어 만화 읽기 |
| 영화 시청자 | ~2,000,000+ | 더빙 영화 시청 |
| 개발자 | ~1,000+ | 워크플로우에 통합 |

### 5.2 경쟁 분석

| 제품 | 강점 | UTH 대비 약점 |
|------|------|---------------|
| Google Translate | 무료, 빠름 | 게임/만화/영화 미지원 |
| DeepL | 고품질 | 텍스트만, 다중 모달 미지원 |
| 수동 번역 | 최고 품질 | 느림, 비쌈 |
| 개별 도구 | 이미 보유 | 통합 없음, TM 공유 없음 |

### 5.3 운영 결론
> **타당** — 거대한 시장, 직접적인 경쟁자 없음. UTH는 최초의 통합 솔루션.

## 6. 종합 평가

| 측면 | 평가 | 신뢰도 |
|------|------|--------|
| 기술 | ✅ 높은 타당성 | 85% |
| 재정 | ✅ 높은 타당성 | 90% |
| 일정 | ✅ 타당 | 80% |
| 운영 | ✅ 높은 타당성 | 85% |
| **종합** | **✅ 수행 권장** | **85%** |

### 권고사항
1. **즉시 착수** — 70% 코드 이미 보유, 통합만 필요
2. **Phase 1 우선** — MiMo API 클라이언트 + Director Agent가 기반
3. **MVP = Game Pipeline** — 가장 빠르고 가치 있음
4. **조기 오픈 소스화** — 커뮤니티가 테스트와 기여에 도움
