## 1. Введение

### 1.1 Назначение

Настоящий документ определяет полную спецификацию требований к программному обеспечению **Universal Translation Hub (UTH)** — мультиагентной AI-системы для автоматического перевода игр, манги и фильмов на вьетнамский язык, построенной на базе платформы Xiaomi MiMo V2.5.

### 1.2 Область применения

UTH обеспечивает автоматизированный перевод медиаконтента через три конвейера:

| Конвейер | Входные данные | Выходные данные |
|----------|---------------|-----------------|
| **Игры** | Файлы движков (12 типов) | Переведённые ресурсы игры |
| **Манга** | Изображения страниц | Переведённые изображения |
| **Фильмы** | Видеофайлы | Видео с дубляжом на вьетнамский |

### 1.3 Определения, акронимы и сокращения

| Термин | Определение |
|--------|-------------|
| **UTH** | Universal Translation Hub |
| **MiMo** | Xiaomi MiMo — AI-платформа компании Xiaomi |
| **OCR** | Optical Character Recognition — распознавание текста |
| **STT** | Speech-to-Text — распознавание речи |
| **TTS** | Text-to-Speech — синтез речи |
| **LLM** | Large Language Model — большая языковая модель |
| **VRAM** | Video RAM — видеопамять GPU |
| **CUDA** | Compute Unified Device Architecture — параллельные вычисления NVIDIA |
| **VLC** | VideoLAN Client — медиаплеер с открытым исходным кодом |
| **API** | Application Programming Interface — программный интерфейс приложения |

## 2. Общее описание

### 2.1 Перспектива продукта

UTH — это автономная система, работающая на локальном оборудовании пользователя. Система интегрирует 10 AI-субагентов, управляемых центральным директором, для обработки медиаконтента в трёх доменах.

```
┌─────────────────────────────────────────────────────────┐
│                   Universal Translation Hub              │
│                                                         │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐              │
│  │  Игры    │  │  Манга   │  │  Фильмы  │  ← Вход     │
│  └────┬─────┘  └────┬─────┘  └────┬─────┘              │
│       │              │              │                    │
│  ┌────▼──────────────▼──────────────▼────┐              │
│  │         Director Agent (Центр.)        │              │
│  └────┬──────────────┬──────────────┬────┘              │
│       │              │              │                    │
│  ┌────▼────┐   ┌────▼────┐   ┌────▼────┐              │
│  │ Router  │   │Translator│   │   QA    │              │
│  └────┬────┘   └────┬────┘   └────┬────┘              │
│       │              │              │                    │
│  ┌────▼────┐   ┌────▼────┐   ┌────▼────┐              │
│  │  OCR    │   │  STT    │   │  TTS    │              │
│  │Inpaint  │   │ Voice   │   │  Voice  │              │
│  │ Render  │   │ VoiceCl │   │  Clone  │              │
│  └─────────┘   └─────────┘   └─────────┘              │
│                                                         │
│  ┌─────────────────────────────────────┐               │
│  │    Xiaomi MiMo V2.5 (8 моделей)     │               │
│  └─────────────────────────────────────┘               │
└─────────────────────────────────────────────────────────┘
```

### 2.2 Функции продукта

| Функция | Описание |
|---------|----------|
| F-01: Распознавание формата | Автоопределение типа входного контента |
| F-02: Маршрутизация | Выбор конвейера обработки |
| F-03: Перевод текста | Перевод на вьетнамский с учётом контекста |
| F-04: OCR | Распознавание текста на изображениях |
| F-05: Инпейнтинг | Удаление оригинального текста с изображений |
| F-06: Рендер | Наложение переведённого текста |
| F-07: STT | Распознавание речи из видео |
| F-08: TTS | Синтез речи на вьетнамском |
| F-09: Клонирование голоса | Сохранение тембра оригинального голоса |
| F-10: QA | Контроль качества перевода |

### 2.3 Характеристики пользователей

| Роль | Описание | Уровень технической подготовки |
|------|----------|-------------------------------|
| **Переводчик** | Основной пользователь, запускает перевод | Средний |
| **Администратор** | Настраивает систему, управляет моделями | Высокий |
| **Разработчик** | Расширяет движки и конвейеры | Высокий |

### 2.4 Ограничения

- Система работает только на GPU NVIDIA с поддержкой CUDA
- Минимум 8 ГБ VRAM (рекомендуется 16+ ГБ)
- Поддержка только Windows 11 и Linux (Ubuntu 22.04+)
- Все AI-модели работают локально (без облачных зависимостей)
- База данных: SQLite (однопользовательский режим)

### 2.5 Допущения и зависимости

- Пользователь имеет стабильное интернет-соединение для загрузки моделей
- GPU-драйверы NVIDIA и CUDA установлены корректно
- FFmpeg доступен в системном PATH
- VLC установлен для предпросмотра видео

## 3. Функциональные требования

### FR-01: Распознавание типа входного контента

| Поле | Значение |
|------|----------|
| **Идентификатор** | FR-01 |
| **Приоритет** | Критический |
| **Описание** | Система должна автоматически определять тип входного файла (игра, манга, видео) |

**Приёмочные критерии:**
- AC-01: Система определяет формат игры из 12 поддерживаемых движков с точностью >= 95%
- AC-02: Система распознаёт изображения манги (JPEG, PNG, WebP, TIFF) с точностью >= 99%
- AC-03: Система распознаёт видеофайлы (MP4, MKV, AVI, MOV, WebM) с точностью >= 99%
- AC-04: При неизвестном формате система возвращает ошибку с описанием

### FR-02: Маршрутизация к конвейеру

| Поле | Значение |
|------|----------|
| **Идентификатор** | FR-02 |
| **Приоритет** | Критический |
| **Описание** | Router Agent должен направлять задачу в соответствующий конвейер |

**Приёмочные критерии:**
- AC-01: Игровые файлы → конвейер Games
- AC-02: Изображения манги → конвейер Manga
- AC-03: Видеофайлы → конвейер Movies
- AC-04: Маршрутизация занимает < 100 мс

### FR-03: Извлечение текста из игр

| Поле | Значение |
|------|----------|
| **Идентификатор** | FR-03 |
| **Приоритет** | Высокий |
| **Описание** | Система должна извлекать текстовые ресурсы из файлов 12 игровых движков |

**Поддерживаемые движки:**

| ID | Движок | Форматы файлов |
|----|--------|----------------|
| E-01 | Ren'Py | .rpy, .rpyc |
| E-02 | RPG Maker | .json, .rvdata2 |
| E-03 | Unity | .txt, .json, .xml |
| E-04 | Unreal Engine | .locres, .po |
| E-05 | NScripter | .txt (nsa/ns2) |
| E-06 | KiriKiri | .ks, .tjs |
| E-07 | YU-RIS | .ybn, .mes |
| E-08 | CatSystem2 | .cst, .cs2 |
| E-09 | TyranoBuilder | .tyrano |
| E-10 | Construct | .json |
| E-11 | Godot | .tres, .translation |
| E-12 | Сustom/Text | .txt, .csv, .xml, .json |

**Приёмочные критерии:**
- AC-01: Корректное извлечение текста из каждого движка с точностью >= 98%
- AC-02: Сохранение структуры (имя персонажа, диалог, системный текст)
- AC-03: Поддержка Unicode и кодировки UTF-8

### FR-04: Распознавание текста на изображениях (OCR)

| Поле | Значение |
|------|----------|
| **Идентификатор** | FR-04 |
| **Приоритет** | Высокий |
| **Описание** | OCR Agent должен распознавать текст на страницах манги |

**Приёмочные критерии:**
- AC-01: Распознавание японского, корейского, английского текста с точностью >= 95%
- AC-02: Определение bounding box каждого текстового блока
- AC-03: Обработка вертикального японского текста
- AC-04: Скорость обработки: <= 5 сек на страницу (при GPU)

### FR-05: Удаление оригинального текста (Инпейнтинг)

| Поле | Значение |
|------|----------|
| **Идентификатор** | FR-05 |
| **Приоритет** | Высокий |
| **Описание** | Inpainting Agent должен удалять оригинальный текст с изображений манги |

**Приёмочные критерии:**
- AC-01: Полное удаление текста без видимых артефактов (PSNR >= 30 dB)
- AC-02: Сохранение фоновой графики и линий рисунка
- AC-03: Обработка пузырей диалогов различных форм
- AC-04: Время обработки: <= 8 сек на страницу

### FR-06: Наложение переведённого текста (Рендер)

| Поле | Значение |
|------|----------|
| **Идентификатор** | FR-06 |
| **Приоритет** | Высокий |
| **Описание** | Render Agent должен накладывать вьетнамский текст на изображения |

**Приёмочные критерии:**
- AC-01: Автоматический подбор размера шрифта под размер пузыря
- AC-02: Поддержка вертикального и горизонтального расположения
- AC-03: Антиалиасинг текста (сглаживание)
- AC-04: Время рендеринга: <= 2 сек на страницу

### FR-07: Перевод текста

| Поле | Значение |
|------|----------|
| **Идентификатор** | FR-07 |
| **Приоритет** | Критический |
| **Описание** | Translator Agent должен переводить текст на вьетнамский язык |

**Приёмочные критерии:**
- AC-01: Поддержка перевода с EN, JA, KO, ZH на VI
- AC-02: Учёт контекста (имена персонажей, терминология)
- AC-03: BLEU-оценка >= 0.70 для общего текста
- AC-04: BLEU-оценка >= 0.80 для игрового диалога
- AC-05: Скорость: >= 100 токенов/сек на GPU

### FR-08: Распознавание речи (STT)

| Поле | Значение |
|------|----------|
| **Идентификатор** | FR-08 |
| **Приоритет** | Высокий |
| **Описание** | STT Agent должен распознавать речь из аудиодорожки фильмов |

**Приёмочные критерии:**
- AC-01: WER (Word Error Rate) <= 10% для чистого аудио
- AC-02: WER <= 20% для аудио с фоновым шумом
- AC-03: Поддержка сегментации по диалогам
- AC-04: Привязка таймкодов к каждому сегменту
- AC-05: Скорость: >= 10x realtime

### FR-09: Синтез речи (TTS)

| Поле | Значение |
|------|----------|
| **Идентификатор** | FR-09 |
| **Приоритет** | Высокий |
| **Описание** | TTS Agent должен генерировать вьетнамскую речь |

**Приёмочные критерии:**
- AC-01: MOS (Mean Opinion Score) >= 3.5/5.0
- AC-02: Поддержка вьетнамского языка с тонами (6 тонов)
- AC-03: Синхронизация по таймкодам оригинала
- AC-04: Регулировка скорости речи
- AC-05: Скорость генерации: >= 5x realtime

### FR-10: Клонирование голоса

| Поле | Значение |
|------|----------|
| **Идентификатор** | FR-10 |
| **Приоритет** | Средний |
| **Описание** | VoiceClone Agent должен клонировать тембр голоса оригинального актёра |

**Приёмочные критерии:**
- AC-01: Клонирование на основе 10+ секунд аудио
- AC-02: Сохранение тональных характеристик голоса
- AC-03: Поддержка мужских и женских голосов
- AC-04: Сходство голоса >= 80% (по субъективной оценке)

### FR-11: Контроль качества (QA)

| Поле | Значение |
|------|----------|
| **Идентификатор** | FR-11 |
| **Приоритет** | Высокий |
| **Описание** | QA Agent должен проверять качество перевода и выявлять ошибки |

**Приёмочные критерии:**
- AC-01: Проверка орфографии вьетнамского текста
- AC-02: Проверка целостности (все ли строки переведены)
- AC-03: Проверка форматирования (теги, спецсимволы)
- AC-04: Генерация отчёта качества с метриками

| gRPC (опционально) | Высокопроизводительная связь между агентами |

## 6. Требования к данным

### 6.1 Логическая модель данных

```sql
-- Таблица проектов
CREATE TABLE projects (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    name        TEXT NOT NULL,
    type        TEXT CHECK(type IN ('game', 'manga', 'movie')),
    source_lang TEXT NOT NULL DEFAULT 'ja',
    target_lang TEXT NOT NULL DEFAULT 'vi',
    status      TEXT DEFAULT 'created',
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at  DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- Таблица задач
CREATE TABLE tasks (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    project_id  INTEGER REFERENCES projects(id),
    agent       TEXT NOT NULL,
    status      TEXT DEFAULT 'pending',
    input_data  TEXT,
    output_data TEXT,
    started_at  DATETIME,
    finished_at DATETIME,
    error_msg   TEXT
);

-- Таблица переводов
CREATE TABLE translations (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    project_id  INTEGER REFERENCES projects(id),
    source_text TEXT NOT NULL,
    trans_text  TEXT,
    context     TEXT,
    confidence  REAL,
    agent       TEXT,
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- Таблица медиафайлов
CREATE TABLE media_files (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    project_id  INTEGER REFERENCES projects(id),
    file_path   TEXT NOT NULL,
    file_type   TEXT,
    file_size   INTEGER,
    checksum    TEXT,
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 6.2 Требования к хранению данных

| Тип данных | Объём | Срок хранения |
|-----------|-------|---------------|
| AI-модели (MiMo V2.5) | ~40 ГБ | Постоянно |
| Входные файлы | Зависит от проекта | До удаления проекта |
| Промежуточные результаты | ~2x размера входа | До очистки кэша |
| База данных SQLite | < 1 ГБ | Постоянно |
| Логи | <= 500 МБ | 30 дней (ротация) |

### 6.3 Требования к миграции данных

- Экспорт проекта в ZIP-архив (исходные + переведённые файлы + БД)
- Импорт проекта из архива
- Совместимость версий БД (миграции через Alembic)
