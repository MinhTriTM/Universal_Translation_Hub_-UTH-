## 1. 引言

### 1.1 目的

本文档为 **Universal Translation Hub (UTH)** 的软件需求规格说明书，遵循 IEEE 830-1998 标准。文档详细描述了系统的功能需求、非功能需求、接口需求和数据需求，作为系统设计、开发和验收的基准。

### 1.2 范围

UTH 是一个多智能体 AI 系统，可自动将 3 种多媒体内容翻译成越南语：

- **游戏翻译**：支持 12 种游戏引擎（RPG Maker、Unity、Ren'Py、Kirikiri 等）
- **漫画翻译**：OCR 文字识别 → 文字修复 → 翻译 → 文字渲染
- **影视配音**：语音识别 → 翻译 → TTS 语音合成 → 角色配音

### 1.3 定义、缩略语和缩写

| 术语 | 定义 |
|------|------|
| UTH | Universal Translation Hub — 通用翻译中心 |
| MiMo | 小米 MiMo V2.5 AI 模型系列 |
| Agent | AI 智能体，执行特定任务的自治单元 |

| TTS | Text-to-Speech — 文字转语音 |

| TER | Translation Edit Rate — 翻译编辑率 |
| BLEU | Bilingual Evaluation Understudy — 双语评估替代指标 |

## 2. 总体描述

### 2.1 产品视角

UTH 是一个独立的桌面应用程序，基于 3 个已有开源项目构建，使用小米 MiMo V2.5 作为核心 AI 引擎。系统采用多智能体架构，10 个 AI 子智能体协同工作，通过统一的 Web UI 提供服务。

```
┌─────────────────────────────────────────────────────────────┐
│                    用户界面层 (FastAPI + Web UI)              │
├─────────────────────────────────────────────────────────────┤
│                    智能体编排层 (Director Agent)              │
├──────────┬──────────────┬──────────────┬────────────────────┤
│ 游戏管道 │   漫画管道   │   影视管道   │     共享服务层      │
│ (12引擎) │ (OCR+修复)   │  (STT+TTS)  │ (翻译/TM/术语/语音)│
├──────────┴──────────────┴──────────────┴────────────────────┤
│                 MiMo V2.5 API 层 (8个模型)                   │
├─────────────────────────────────────────────────────────────┤
│              基础设施层 (SQLite/FFmpeg/VLC/文件系统)          │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 产品功能

| 编号 | 功能 | 管道 | 描述 |
|------|------|------|------|
| F-01 | 游戏文本提取 | 游戏 | 从 12 种游戏引擎中提取可翻译文本 |
| F-02 | 游戏文本注入 | 游戏 | 将翻译后的文本写回游戏文件 |
| F-03 | 漫画 OCR | 漫画 | 识别漫画图片中的文字区域 |
| F-04 | 图像修复 | 漫画 | 移除原文文字，恢复背景 |
| F-05 | 文字渲染 | 漫画 | 将翻译文字渲染到修复后的图片 |
| F-06 | 语音识别 | 影视 | 从视频音轨中提取文字（STT） |
| F-07 | 语音合成 | 影视 | 将翻译文字转为语音（TTS） |
| F-08 | 声音克隆 | 影视 | 克隆原始角色声音特征 |
| F-09 | 多语言翻译 | 共享 | 中/日/韩/英 → 越南语翻译 |
| F-10 | 翻译记忆 | 共享 | 存储和复用历史翻译 |
| F-11 | 术语管理 | 共享 | 统一术语翻译 |
| F-12 | 质量检查 | 共享 | 翻译质量自动评估 |

### 2.3 用户特征

| 用户类型 | 描述 | 技术水平 |
|----------|------|----------|
| 游戏汉化组成员 | 将日/中/韩游戏翻译成越南语 | 中等，熟悉游戏文件结构 |
| 漫画翻译者 | 翻译漫画图片 | 中等，熟悉图像编辑 |

| 约束 | 描述 |

### 6.2 数据库模式（核心表）

```sql
-- 翻译单元表
CREATE TABLE translation_units (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    project_id INTEGER NOT NULL,
    source_lang VARCHAR(10) NOT NULL,
    target_lang VARCHAR(10) DEFAULT 'vi',
    source_text TEXT NOT NULL,
    target_text TEXT,
    context TEXT,
    status VARCHAR(20) DEFAULT 'pending',
    quality_score FLOAT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (project_id) REFERENCES projects(id)
);

-- 翻译记忆库
CREATE TABLE translation_memory (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    source_lang VARCHAR(10) NOT NULL,
    target_lang VARCHAR(10) NOT NULL,
    source_text TEXT NOT NULL,
    target_text TEXT NOT NULL,
    domain VARCHAR(50),
    confidence FLOAT DEFAULT 1.0,
    hit_count INTEGER DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 术语表
CREATE TABLE terminology (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    term VARCHAR(200) NOT NULL,
    translation VARCHAR(200) NOT NULL,
    source_lang VARCHAR(10) NOT NULL,
    domain VARCHAR(50),
    notes TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 声音配置文件
CREATE TABLE voice_profiles (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    character_name VARCHAR(100) NOT NULL,
    model_type VARCHAR(50) NOT NULL,
    config_json TEXT NOT NULL,
    sample_audio_path VARCHAR(500),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 项目表
CREATE TABLE projects (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name VARCHAR(200) NOT NULL,
    pipeline_type VARCHAR(20) NOT NULL,
    source_lang VARCHAR(10) NOT NULL,
    status VARCHAR(20) DEFAULT 'active',
    config_json TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```
